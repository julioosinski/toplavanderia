Nome da pasta (na raiz do projeto, ao lado de package.json):

  secrets

Nome do FICHEIRO onde cola o token (uma linha só, sem aspas), por ordem de leitura:

  1) cli-access-token   (recomendado se "supabase-access-token" for uma pasta presa no Windows)
  2) supabase-access-token
  3) supabase-access-token.txt

IMPORTANTE:
  supabase-access-token deve ser um FICHEIRO, nao uma pasta.
  Se o Explorer criou uma pasta com esse nome, apague a pasta,
  copie o .example para um novo ficheiro com esse nome e cole o token.

Caminho completo:

  secrets/supabase-access-token

Como criar no Windows (PowerShell, na raiz do projeto):

  Copy-Item secrets\supabase-access-token.example secrets\supabase-access-token

Depois abra secrets\supabase-access-token no editor e substitua o texto pelo token real.

O comando "pnpm run supabase:db:push" tambem aceita se o token estiver
dentro da primeira pasta errada secrets\supabase-access-token\, mas o ideal e corrigir para um so ficheiro.
